IP and HTTP

We briefly mentioned IP addresses earlier, but what exactly is an IP address made of? For starters, it is a collection of numbers that is unique to any device on a network. IP addresses come in parts, just like a street address. The parts consist of the country/network, the region/network, the subnetwork, and the device itself. This format is called IPv4 and each of the parts is represented by a portion of numbers. Each part is made up of 8 bits, totalling 32 bits for the entire address. Computers need to use IP addresses in order to send data between one another. Similarly to sending a letter in the mail, a computer will reach out to another computer by sending data and leaving behind its original address so that the receiving computer knows where to send data back.


HTTP is one of the major internet protocols that exist today. A protocol is just a set of rules that we use. HTTP stands for Hyper Text Transfer Protocol, and it basically is a set of rules for how to communicate on the internet. This doesn't mean that users have defined rules in how they can communicate on the internet, but instead how data transmitted over a network communicates with other computers and networks. In HTTP, users can send text and the computer will convert that into binary zeros and ones. When the data is received, it is immediately interpreted and translated back into its original text. You can even transfer other types of data such as images or sound. Think of HTTP as an interpreter that translates from text to binary and then back to text again. Using HTTP, it allows computers to communicate to each other and the people using the computers to communicate with each other as well. Since computers do not understand language that people use, it is important that data is translated into what computers do understand, to ensure proper transfer of data from one person to another.
